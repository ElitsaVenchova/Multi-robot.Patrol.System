Patrol and attack behaviors are implemented using the Strategy design pattern, allowing interchangeable and extensible behavior definitions.

The simulation engine interprets the DSL configuration and executes the simulation through a discrete-time loop, where patrol strategies and attack models interact within the defined environment.

"interpretation of DSL constructs into executable simulation components"

The simulation engine interprets the DSL configuration and executes the simulation through a discrete-time loop, where patrol strategies and attack models interact within the defined environment.

The results module computes performance metrics such as interception rate and detection efficiency, enabling systematic evaluation and comparison of patrol strategies.

Simulation results are exported in a structured format (CSV), enabling further statistical analysis and visualization of performance metrics.

In addition to interception rate, we evaluate the temporal efficiency of patrol strategies by measuring the average detection time, defined as the delay between attack occurrence and interception.

Each configuration was executed multiple times, and the results are reported as mean values with standard deviation to account for stochastic variability.

Patrol strategies are configured through structured DSL elements that support parameterization, enabling flexible experimentation with strategy-specific configurations such as robot distribution.

As the attack rate increases, the success rate decreases while detection time increases, indicating higher system load and reduced responsiveness.

A parallel execution model was explored to evaluate the scalability of the simulation, where robot behaviors and detection processes are executed concurrently using a thread pool.

To ensure reproducibility, all stochastic components of the simulation are controlled through a centralized random number provider initialized with a configurable seed.

Robot characteristics are defined in separate configuration files, enabling reuse and reducing coupling between simulation scenarios and platform-specific parameters.

The attack model supports both persistent and time-constrained threats, allowing the evaluation of patrol strategies under different temporal conditions.
